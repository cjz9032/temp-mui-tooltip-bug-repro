🐛 MUI Tooltip controlled-mode onClose bug

Tooltip internal guard if (onClose && open) reads the render-time closure open, not the latest state. When React batches setOpen(true) and mouseleave in the same tick, handleClose sees stale open=false → onClose is dropped. This simulates the race condition via dispatchEvent (real MouseEvents, real handlers, real React 18 batching).
